Шпаргалка по GIT

git config --global user.name Добавляємо наш нік.Бажано мати такий самий як на гітхабі.

git config --global user.email blabal@bla "Добавляємо нашу пошту

git init Створюємо базу,ініціалізуєм проєкт

git status Команда показує стан файлів .Таких станів є 3-staged,modified,commited

git add Добавляєм файл в відслідковуєму зону

git commit Cтворюємо коміт,тобто робим знімок стану даного репозиторія

git log Переглядаєм історію комітів

git git checkout Переміщення між комітамиа

git branch Команда яка показує на якій гілці ми зараз перебуваєм

git push Вигружаєм файли з локального репозиторія на віддалений

git pull Команда для отримання оновлень з віддаленого репозиторія

git merge Використовується для злиття гілок

git clone Використовується для клонування віддаленого репозиторія собі на компютер

gitignore файл Використовується тоді коли є файли які ми не хочем щоб git їх індексував...відображає такі файли як неконтрольовані

git add Добавляєм файл в відслідковуєму зону

git reset Команда для відкату назад на певну кількість комітів,яку ми задаєм.Ця команда має три параметри= soft /mixed/ hard

git rebase Команда для отримання оновлень з віддаленого репозиторія

git cherry-pick Команда використовується тоді коли нам потрібно взяти один або кілька комітів з іншої гілки в нашу гілку

git revert Команда для видалення останього коміта,тобто вертаємся до попереднього коміта який був

git stash Ховає зміни в файлах і зберігає ці зміни окремо, щоб потім можна було їх повернути.
